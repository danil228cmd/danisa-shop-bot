# 🔍 Диагностика проблемы с админ панелью

## Проблема: Открывается магазин вместо админки

### Шаг 1: Проверьте URL

Откройте браузер и посмотрите на адресную строку:

**Если видите:**
- `https://ваш-сайт.railway.app/` → ЭТО МАГАЗИН!
- `https://ваш-сайт.railway.app/miniapp/` → ЭТО ТОЖЕ МАГАЗИН!

**Нужно открыть:**
- `https://ваш-сайт.railway.app/admin/` → ЭТО АДМИНКА!

### Шаг 2: Проверьте заголовок страницы

Посмотрите на вкладку браузера:

- ✅ Если написано **"🔐 АДМИН ПАНЕЛЬ"** → это админка
- ❌ Если написано **"DANISA SHOP"** → это магазин

### Шаг 3: Проверьте содержимое страницы

**Админ панель должна показывать:**
- Форму входа с паролем
- Заголовок "🔐 АДМИН-ПАНЕЛЬ"
- После входа: вкладки "Категории", "Подкатегории", "Товары", "Заказы"

**Магазин показывает:**
- Каталог товаров
- Категории товаров
- Корзину

### Шаг 4: Тест локально

Запустите локально и проверьте:

```bash
# Запустите сервер
node server.js

# Откройте в браузере ДВА окна:
```

**Окно 1:** http://localhost:3000/admin/
- Должна открыться админка

**Окно 2:** http://localhost:3000/
- Должен открыться магазин

Если оба окна показывают одно и то же → проблема в коде!

### Шаг 5: Проверьте файлы

Убедитесь что файлы на месте:

```bash
dir public\admin\index.html
dir public\miniapp\index.html
```

Оба файла должны существовать!

### Шаг 6: Проверьте Railway

Если локально работает, но на Railway нет:

1. Откройте Railway → ваш проект
2. Перейдите в **Deployments**
3. Выберите последний деплой
4. Нажмите **View Logs**
5. Найдите строку: `Сервер запущен на PORT`

Если ошибок нет, значит проблема в URL!

## 🎯 Быстрое решение

### Вариант 1: Используйте правильный URL

Просто добавьте `/admin/` в конец:

```
https://ваш-сайт.railway.app/admin/
```

### Вариант 2: Используйте Telegram бота

1. Откройте бота
2. Отправьте `/admin`
3. Нажмите кнопку

### Вариант 3: Запустите тест локально

```bash
# Windows
test-admin.bat

# Или вручную
node server.js
# Затем откройте: http://localhost:3000/admin/
```
